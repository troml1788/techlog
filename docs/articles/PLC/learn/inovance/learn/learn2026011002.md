### **知识点归纳总结**

#### **一、 本次讲解的计时器特点、用法和实际应用场景**

| 定时器类型 | 核心特点 | 执行过程与用法 | 典型应用场景 |
| :--- | :--- | :--- | :--- |
| **关断延时定时器 (TOFR)** | **断电延时**：在输入信号**断开（OFF）** 的瞬间开始计时。 | 1. **IN为ON**：Q立即为ON，不计时。<br>2. **IN由ON变OFF**：开始计时，Q保持ON。<br>3. **计时中**：若IN重新变ON，则**ET立即清零**，Q保持ON。<br>4. **ET ≥ PT**：Q变为OFF。<br>5. **复位(R)**：任何时刻，R信号都能强制Q为OFF且ET清零。 | 1. **设备延时停止**：如风扇、水泵的延时关闭。<br>2. **信号滞后响应**：确保主设备先停，辅助设备后停（如逆停）。 |
| **时间累加定时器 (TACR)** | **累加/保持**：计时可暂停和继续，当前值（ET）具有断电保持功能。 | 1. **IN为ON**：ET从0或保持值开始累加。<br>2. **IN为OFF**：计时暂停，ET保持当前值。<br>3. **IN再次为ON**：ET从上次保持的值继续累加。<br>4. **ET ≥ PT**：Q变为ON。<br>5. **复位(R)**：只有R信号能清零ET和复位Q。 | 1. **设备累计运行时间**统计（如计算保养周期）。<br>2. **故障总时间累计**。适用于需要断续、分时段计时的场景。 |

**核心区别补充**：
*   **与TONR的区别**：TACR的计时可随时暂停/继续；而TONR在IN为ON时计时，IN为OFF时**保持但不复位**ET，下次IN为ON时**不会继续累加**（除非ET被复位）。
*   **记忆性**：TACR的ET值具有断电保持功能，是真正的“累计”，而普通定时器断电后数据会丢失。

#### **二、 “顺起逆停”电路设计思路**

**1. 控制要求：**
*   **启动顺序**：水泵（Y1）启动 → 延时5秒 → 电机（Y0）启动。
*   **停止顺序**：电机（Y0）停止 → 延时3秒 → 水泵（Y1）停止。

**2. 实现思路与定时器选择：**
整个控制逻辑分为“启动”和“停止”两个独立的阶段。

*   **启动阶段（“顺起”）**：
    *   **目标**：实现一个动作（启动电机）滞后于另一个动作（启动水泵）。
    *   **方案**：使用**接通延时定时器（TON）**。利用水泵启动后触发的TON定时器，延时5秒后再去启动电机。这是最直观的延时启动方法。

*   **停止阶段（“逆停”）**：
    *   **目标**：实现一个动作（停止水泵）滞后于另一个动作（停止电机）。
    *   **方案**：使用**关断延时定时器（TOFR）**。这是本案例的精髓。
        *   当发出停止信号（如按下X1）时，**立即复位电机（Y0）**。
        *   利用**电机（Y0）的常开触点**作为TOFR定时器的输入（IN）。当Y0从ON（运行）变为OFF（停止）时，触发了TOFR的计时条件。
        *   TOFR开始计时3秒，计时期间其输出触点保持接通，从而维持水泵（Y1）的运行。
        *   3秒时间到，TOFR输出断开，**再复位水泵（Y1）**。

**3. 核心逻辑框图：**
```
启动流程：
X0(启动按钮) → 置位Y1(水泵运行) → TON(开始5秒计时) → 计时到 → 置位Y0(电机运行)

停止流程：
X1(停止按钮) → 复位Y0(电机停止) → [Y0常开触点断开] → 触发TOFR(开始3秒计时) → 计时到 → 复位Y1(水泵停止)
```

**总结**：“顺起逆停”电路的关键在于根据不同的控制阶段（启动/停止）和延时需求（延时接通/延时断开），灵活选择最合适的定时器（TON用于延时接通，TOFR用于延时断开），并通过中间继电器（M点）清晰地进行逻辑状态传递，从而构建出稳定可靠的顺序控制逻辑。
